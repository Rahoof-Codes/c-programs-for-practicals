1.Function Overloading & Inline Functions

Common Errors:

  1. Placement: Forgetting that inline functions must be defined before they are called in main(). 

#include <iostream.h>
#include <conio.h>

inline int square(int x)
{
    return x * x;
}

int add(int a, int b = 10)
{
    return a + b;
}

int add(int a, int b, int c)
{
    return a + b + c;
}

void main()
{
    clrscr();
    cout << "Square of 5: " << square(5) << endl;
    cout << "Add (5 + default): " << add(5) << endl;
    cout << "Add (5 + 6 + 7): " << add(5, 6, 7) << endl;
    getch();
}

2.Class and Objects

Common Errors:

  1. Don't forget the semicolon (;) after the closing curly bracket } of the class. Also, remember that you cannot access private variables directly inside main().
​ 
#include <iostream.h>
#include <conio.h>
​class Student {
public:
int roll;
void getData() {
roll = 101;
}
void showData() {
cout << "Roll Number: " << roll;
}
};
​void main() {
clrscr();
Student s; // Object creation
s.getData();
s.showData();
getch();
}

3.Passing Objects to Functions

Common Errors:

  1. If you pass an object normally, the function creates a copy of it. Any changes you make inside that function will not happen to the original object in main().
​ 
#include <iostream.h>
#include <conio.h>
​class Number {
public:
int a;
};
​void display(Number n) { 
cout << "Value: " << n.a;
}
​void main() {
clrscr();
Number obj;
obj.a = 50;
display(obj);
getch();
}

4.Friend Functions

Common Errors:

  1. A common mistake is forgetting the keyword friend inside the class. Without it, the function is treated as a stranger and cannot see the private data. 

#include <iostream.h>
#include <conio.h>
​class Sample {
int x;
public:
Sample() { x = 20; }
friend void show(Sample s);
};
​void show(Sample s) {
cout << "Value of x: " << s.x;
}
​void main() {
clrscr();
Sample obj;
show(obj);
getch();
}

5.Concept of Passing Objects

Common Errors:

  1. In this method, the original value does not change. Students often write the code expecting the value in main() to update, but only the copy inside the function updates. 

#include <iostream.h>
#include <conio.h>
​class Test {
public:
int x;
};
​void change(Test t) {
t.x = 100;
cout << "Inside function: " << t.x << endl;
}
​void main() {
clrscr();
Test obj;
obj.x = 10;
change(obj);
cout << "Outside function: " << obj.x;
getch();
}

6.Constructor and Destructor

Common Errors:

  1. A constructor must have the same name as the class and no return type (not even void). Also, never try to pass arguments to a Destructor (~Demo()). 

#include <iostream.h>
#include <conio.h>
​class Demo {
public:
Demo() {
cout << "Constructor called" << endl;
}
~Demo() {
cout << "Destructor called";
}
};
​void main() {
clrscr();
Demo obj;
getch();
}

7.Unary Operator Overloading

Common Errors:

  1. You must declare the operator function properly inside the class. If you are overloading ++ for postfix (like a++), you need to add a dummy int argument to distinguish it. 

​#include  <iostream.h>
#include <conio.h>
​class Unary {
int x;
public:
Unary(int a) { x = a; }
void operator ++() {
x = x + 1;
}
void display() {
cout << "Value = " << x;
}
};
​void main() {
clrscr();
Unary obj(10);
++obj;
obj.display();
getch();
}

8.Binary Operator Overloading

Common Errors:

  1. When overloading + inside a class, you only need one argument (the object on the right). Students often mistakenly try to pass two arguments 

​#include <iostream.h>
#include <conio.h>
​class Add {
int a;
public:
Add(int x) { a = x; }
Add operator +(Add obj) {
return Add(a + obj.a);
}
void display() {
cout << a;
}
};
​void main() {
clrscr();
Add a1(10), a2(20);
Add a3 = a1 + a2;
cout << "Sum is: ";
a3.display();
getch();
}

9.A)Inheritance (Single)

Common Errors:

  1. if you forget to write public (e.g., class B : public A), the inheritance defaults to private. This means main() won't be able to use the functions from Class A. 

​#include <iostream.h>
#include <conio.h>
​class A {
public:
void showA() { cout << "Class A" << endl; }
};
​class B : public A {
public:
void showB() { cout << "Class B"; }
};
​void main() {
clrscr();
B obj;
obj.showA();
obj.showB();
getch();
}

9.B)Multilevel Inheritance

Common Errors:

  1. A common error is not checking the order of inheritance (A to B, then B to C). If you try to access a member from 'A' using an object of 'B' that inherited privately, it will fail. 

​#include <iostream.h>
#include <conio.h>
​class A {
public:
void showA() { cout << "Class A" << endl; }
};
​class B : public A {
public:
void showB() { cout << "Class B" << endl; }
};
​class C : public B {
public:
void showC() { cout << "Class C"; }
};
​void main() {
clrscr();
C obj;
obj.showA();
obj.showB();
obj.showC();
getch();
}

10.Virtual Function

Common Errors:

  1. Virtual functions only work if you use a pointer to the base class (e.g., Base *b). If you just use normal objects, the program will ignore the "virtual" logic 

​#include <iostream.h>
#include <conio.h>
​class Base {
public:
virtual void show() {
cout << "Base Class";
}
};
​class Derived : public Base {
public:
void show() {
cout << "Derived Class";
}
};
​void main() {
clrscr();
Base *b;
Derived d;
b = &d;
b->show();
getch();
}

11.Text File Heading

Common Errors:

  1. The most common error is forgetting to close the file with file.close(). If you don't close it, the data you wrote might not be saved properly on the disk. 

​#include <iostream.h>
#include <fstream.h>
#include <conio.h>

void main() {
clrscr();
ofstream file("test.txt");
file << "Hello File";
file.close();

ifstream infile("test.txt");
char text[20];
infile >> text;
cout << text;
infile.close();
getch();
}

13.Biggest Number(Command Line Arguments)

Common Errors:

  1. The program will crash or give an error if you forget to type the numbers in the Run > Arguments box before starting. argv[] needs that input to work 

​#include <iostream.h>
#include <stdlib.h>
#include <conio.h>

void main(int argc, char *argv[]) {
clrscr();
int a = atoi(argv[1]);
int b = atoi(argv[2]);
cout << (a > b ? a : b);
getch();
}

14.Class Template

Common Errors:

  1. When declaring the object in main(), you must specify the data type in brackets (e.g., Test obj). If you forget , the compiler won't know what type of data to store.. 

​#include <iostream.h>
#include <conio.h>

template <class T>
class Test {
T a;
public:
Test(T x) { a = x; }
void show() { cout << a; }
};

void main() {
clrscr();
Test<int> t(100);
t.show();
getch();
}

15.Function Template

Common Errors:

  1. If you try to pass two different types (like one int and one float) to a template that expects only one type T, the compiler will give a mismatch error. 

​#include <iostream.h>
#include <conio.h>

template <class T>
T add(T a, T b) {
return a + b;
}

void main() {
clrscr();
cout << add(10, 20);
getch();
}
